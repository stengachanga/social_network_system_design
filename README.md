# System Design социальной сети для курса по System Design

Ссылка на курс: [platform-balun.ru](https://platform-balun.ru/teach/control/stream/view/id/934232142)

## 1.Функциональные требования:

### 1.1 Публикация постов из путешествий с фотографиями, небольшим описанием и привязкой к конкретному месту путешествия;

- возможность прикрепить одну или несколько фотографий
- возможность указать место путешествия через плагин Яндекс.Карты
- возможность редактировать:
  - фотографии
  - описание
  - местоположение
- возможность удалить пост

### 1.2 Оценка и комментарии постов других путешественников
**Роли:**
- "Автор" - создатель поста, не может ставить оценку
- "Пользователь" - остальные пользователи

**Оценка и комментарии:**
- возможность всем прикрепить к объекту "Пост" неограниченное кол-во текстовых комментариев
- для роли "Пользователь":
  - возможность поставить одну оценку от 1 до 5
  - возможность изменения оценки
- в текстовых комментариях:
  - возможность вставлять эмодзи
  - возможность вставлять фотографии
  - возможность вставлять gif
  - возможность ставить реакции на основе набора эмодзи

### 1.3 Подписка на других путешественников, чтобы следить за их активностью

**Подписка/отписка:**
- для пользователей с ролью "Пользователь" к объекту "Пост" добавляется атрибут "Подписка"/"Отписка"
- подписка действует на все посты автора и добавляет новые посты в ленту с подписками
- возможность получения уведомлений через push о новых постах автора

### 1.4 Поиск популярных мест для путешествий и просмотр постов с этих мест

**Вкладка "Популярное":**
- фильтрация постов по региону
- возможность открыть плагин Яндекс.Карты с возможностью просмотра популярных мест:
  - средняя оценка >4.5
  - количество оценок от 500

### 1.5 Просмотр ленты других путешественников и ленты пользователя, основанной на подписках в обратном хронологическом порядке
- просмотр:
  - ленты других путешественников
  - ленты пользователя, основанной на подписках
- порядок: обратный хронологический

**Типы лент:**
1. Общая лента:
   - новые посты
   - фильтрация:
     - "Популярное"
     - "Новое"
2. Лента пользователя:
   - авторы, на которые подписан
3. Лента "Мои посты"

**Профиль пользователя:**
- возможность посмотреть:
  - свои подписки
  - своих подписчиков
- возможность добавить аватар

### 1.6 Дополнительно
- возможность регистрации в системе с заполнением профиля
- возможность поделиться:
  - ссылкой на профиль
  - ссылкой на любой пост во внешней системе

## 2.Нефункциональные требования:

### 2.1Основные метрики
- DAU >10 000 000 через год при равномерном росте
- Аудитория: страны СНГ
- Данные будем хранить всегда

### 2.2 Активность пользователей
| Действие            | Количество в день |
|---------------------|------------------|
| Публикация постов   | 0.1-0.3          |
| Просмотр постов     | 10               |
| Комментарии         | 1                |
| Реакции             | 5                |

### 2.3 Лимиты
**Для постов:**
- фото: до 12 штук
- максимальный размер: до 12 Мб
- описание: до 512 символов

**Для комментариев:**
- текст: до 128 символов

**Ограничения:**
- максимальное количество подписчиков: до 10 млн
- максимальное количество подписок: до 1 тыс

### 2.4 Сезонность
- Летний период: ×5 нагрузка
- Зимние праздники: ×2 нагрузка

### 2.5 Производительность
| Операция             | Максимальное время |
|----------------------|-------------------|
| Публикация поста     | 30 сек            |
| Получение поста      | 1 сек             |
| Публикация комментария | 0.5 сек         |

## 2.6 Оценка нагрузки:

## 2.6.1 Модель данных:

### Объект "Пост"
| Поле         | Размер      |
|--------------|------------|
| id           | 8 byte     |
| author_id    | 8 byte     |
| description  | 1 024 byte |
| attach_foto  | 144 MB     |
| geo_tag      | 20 byte    |
| timestamp    | 8 byte     |
| stars        | 8 byte     |
**Итого:** 145 MB

### Объект "Реакция"
| Поле         | Размер   |
|--------------|---------|
| id           | 8 byte  |
| author_id    | 8 byte  |
| reaction_id  | 4 byte  |
| timestamp    | 8 byte  |
**Итого:** 28 byte

### Объект "Комментарий"
| Поле         | Размер    |
|--------------|----------|
| id           | 8 byte   |
| author_id    | 8 byte   |
| description  | 256 byte |
| attach_foto  | 12 MB    |
| reaction_sum | 8 byte   |
| timestamp    | 8 byte   |
**Итого:** ~12 MB

### RPS (запросов в секунду)
**Write RPS:**
RPS = dau \* avg_requests_per_day_by_user / 86 400 

RPS = 10000000 * (0.3+1+5) / 86 400 = 177

**Raed RPS:**

RPS = 10000000 * 10 / 86 400 = 1157
## Расчет трафика:


**Write трафик:**

traffic = RPS*(0.3*post+1*comment+5*reaction)=10177 Mb/s

**Read трафик:**

traffic = RPS * (10*post) = 11570 Mb/s
